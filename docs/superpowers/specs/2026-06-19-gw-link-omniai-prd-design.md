# GW-LINK OmniAI 产品需求文档

文档版本：V0.1  
文档日期：2026-06-19  
文档类型：业务确认版 PRD  
产品名称：GW-LINK OmniAI  
中文名称：GW-LINK 全域智能创作台

## 1. 项目背景

GW-LINK OmniAI 是一款面向个人创作者和泛内容生产用户的多端 AI 创作产品。产品通过 GW-LINK 已有 AI 中转站统一接入文本对话、图片生成、视频生成等模型能力，帮助用户在桌面端和移动 App 中完成从想法、文案、视觉素材到视频素材的一站式生成。

本项目采用对外商业化产品路线，首版目标不是做内部工具或企业私有化平台，而是优先验证个人用户和创作者用户的付费创作闭环。

## 2. 产品名称与定位

### 2.1 产品名称

英文名称：GW-LINK OmniAI  
中文名称：GW-LINK 全域智能创作台

### 2.2 产品定位

GW-LINK OmniAI 是一款支持 Windows、macOS、Linux、iOS、Android 的多端 AI 创作工具，通过 GW-LINK 自有 AI 中转站统一接入文本对话、图片生成、视频生成等能力，为个人创作者和泛内容生产用户提供稳定、安全、可商业化的一站式 AI 创作入口。

### 2.3 产品标语

一个入口，连接文本、图片、视频等全域 AI 创作能力。

## 3. 产品目标

### 3.1 业务目标

1. 建立 GW-LINK 面向 C 端用户的 AI 创作产品品牌。
2. 通过免费体验、订阅套餐、点数包形成完整商业化闭环。
3. 将文本、图片、视频生成能力统一包装为简单易用的创作工具。
4. 通过多端覆盖提升用户触达、留存和使用频次。
5. 复用 GW-LINK 已有 AI 中转站能力，缩短产品上线周期。

### 3.2 用户目标

1. 用户可以快速完成文案、图片、视频素材生成。
2. 用户不需要理解复杂模型差异，也能使用推荐模型和默认参数开始创作。
3. 用户可以在桌面端进行主要创作，在移动端查看、生成、分享和接收任务通知。
4. 用户可以清楚看到自己的套餐、点数、生成历史和资产。
5. 用户在生成失败时不会产生误扣费，付费体验可靠。

## 4. 目标用户

### 4.1 核心用户

1. 自媒体创作者：需要文案、封面图、短视频素材。
2. 短视频创作者：需要脚本、分镜、图片、视频片段。
3. 电商运营：需要商品文案、活动图、营销视频素材。
4. 设计与内容从业者：需要快速做创意探索和素材生成。
5. AI 工具尝鲜用户：愿意尝试多模型、多模态生成工具。

### 4.2 非首版重点用户

1. 大型企业私有化客户。
2. 需要团队审批、权限、审计的组织客户。
3. 需要专业剪辑、后期包装能力的视频制作团队。
4. 需要训练自有模型的技术用户。

这些用户可以作为 V2.0 及后续商业拓展方向，不作为 MVP 的验收重点。

4. iOS App。

7. 创作资产库。
8. 多端历史同步。

## 6. 核心用户流程

### 6.1 新用户首次体验流程

1. 用户下载桌面端或安装移动 App。
2. 用户使用手机号或邮箱注册登录。
3. 系统发放免费体验额度。
4. 用户选择文本、图片或视频入口。
5. 用户使用推荐模型和默认参数完成首次生成。
6. 生成结果保存到创作资产库。
7. 用户查看剩余额度。
8. 额度不足或使用高级模型时，引导购买订阅或点数包。

### 6.2 付费创作流程

1. 用户登录账号。
2. 用户选择生成类型。
3. 用户输入创作需求。
4. 用户选择模型、比例、规格或时长。
5. 系统展示预计消耗额度。
6. 用户确认生成。
7. 服务端完成额度校验和任务提交。
8. 生成成功后扣减额度并保存资产。
9. 生成失败时退回额度并展示失败原因。

### 6.3 视频异步任务流程

1. 用户输入视频提示词或上传参考图片。
2. 用户选择模型、比例、分辨率、时长和运动描述。
3. 系统提交视频生成任务。
4. 用户可以离开当前页面，任务在后台继续执行。
5. 用户在任务中心查看进度。
6. 任务完成后，桌面端和移动端展示通知。
7. 用户预览、下载、收藏或重新生成。
8. 任务失败时，系统展示失败原因并退回额度。

5. 任务完成后保存到创作资产库。

首版不做专业剪辑器，不要求支持时间线、多轨剪辑、字幕编辑、转场、调色、音频混音等能力。

#### 7.6.1 商业模式

产品采用订阅套餐 + 点数包混合模式。

#### 7.6.2 套餐建议

1. 免费版：提供少量体验额度。

#### 7.6.3 点数包

1. 用户可以在套餐额度不足时购买点数包。
2. 文本、图片、视频按不同规则折算点数。

4. 生成失败不扣费或自动退回点数。

1. 支持购买订阅套餐。
2. 支持购买点数包。
3. 支持查看订单记录。
4. 支持支付成功后自动刷新会员和点数状态。
5. 支持支付失败提示和重试。

## 8. AI 中转站依赖

GW-LINK 已有可用 AI 中转站接口。本项目首版把中转站作为核心依赖，不从零建设完整模型网关。

### 8.1 中转站在本项目中的职责

1. 接收产品端模型调用请求。
2. 调用实际 AI 模型供应商。
3. 返回文本、图片、视频生成结果或任务状态。
4. 隐藏第三方模型供应商密钥。
5. 提供模型调用结果和错误信息。

### 8.2 产品侧需要补齐的能力

1. 登录鉴权。
2. 套餐权益判断。
3. 点数扣减和退回。
4. 生成记录保存。
5. 前台模型展示包装。
6. 用户订单和会员状态。
7. 内容安全提示和用户协议入口。

## 9. 体验原则

### 9.1 简单优先

用户打开产品后应能直接开始创作。默认模型、默认比例、默认参数必须可用，不能要求用户先理解复杂模型配置。

### 9.2 三种生成入口保持一致

文本、图片、视频都遵循类似体验：

1. 输入需求。
2. 选择模型或使用推荐模型。
3. 查看预计消耗。
4. 点击生成。
5. 保存到资产库。

### 9.3 高级能力渐进展示

高级参数、负面提示词、分辨率、时长、运动描述等能力可以提供，但不应干扰新用户完成首次生成。

### 9.4 付费透明

所有可能消耗点数或套餐额度的操作，都需要在用户确认前明确展示预计消耗。失败任务需要明确提示是否退回额度。

## 10. 安全、合规与风控

### 10.1 密钥安全

1. 客户端不保存第三方模型供应商 API Key。
2. 所有模型调用通过 GW-LINK 中转站。
3. 用户请求必须经过登录鉴权。

### 10.2 额度安全

1. 额度扣减在服务端完成。
2. 客户端只展示服务端返回的余额。
3. 生成失败必须触发不扣费或退回逻辑。
4. 支付订单状态以服务端校验结果为准。

### 10.3 内容安全

1. 支持违规内容拦截。
2. 支持违规提示。
3. 支持生成内容免责声明。
4. 支持用户协议、隐私政策、会员协议入口。
5. 支持基础日志审计。

## 14. 风险与应对

### 14.1 多端首发成本较高

风险：桌面三平台和移动双端同时首发，会增加研发、测试、上架和适配成本。  
应对：统一账号、资产、套餐、生成任务等核心能力；各端首版功能保持一致但交互复杂度不同。桌面端承担完整创作，移动端优先承担轻量生成、查看、分享和通知。

### 14.2 视频生成稳定性不确定

风险：视频生成耗时长、失败率高、成本高，容易影响用户付费体验。  
应对：视频必须采用异步任务；生成前展示预计消耗；失败自动退回额度；任务状态清晰展示；后台持续监控失败率。

### 14.3 模型成本影响毛利

风险：不同模型成本差异大，免费额度和低价套餐可能造成亏损。  
应对：建立点数消耗规则；高级模型绑定高阶套餐；免费额度限制视频能力；定期监控单用户生成成本。

### 14.4 中转站能力与产品体验不匹配

风险：已有中转站接口可能不能完全满足多端产品化展示、任务状态、错误提示、计费映射等需求。  
应对：产品侧建立适配层，对中转站响应进行统一包装；需要中转站提供稳定的任务 ID、状态查询和错误码。

## 15. 业务确认结论

GW-LINK OmniAI 首版应按“创作者商业化优先”路线推进。产品以多端 AI 创作为核心卖点，首发覆盖文本、图片、视频三类生成能力，并通过订阅套餐 + 点数包完成商业化闭环。

本 PRD 的确认重点是产品方向、功能范围、首版边界和商业化模式。技术选型、接口细节、数据库设计、客户端架构和实施排期应在后续实施计划中继续拆分。
